dispose()method 
override public function dispose():void

Disposes the display to clean up resources that are no longer needed. A call to this method stops the display and removes any event/signal listeners. You normally don't call this method manually. Instead the parent screen calls it automatically when the screen is being disposed.

If you want to override this method make sure to call super.dispose() at the start of your overriden dispose method.

override protected function enableChildren():void

Used to enable any child display objects. Called whenever the enabled property of the display is set to true.

This is an abstract method. Override it in your sub-display class and enable any child display objects here that should be enabled when the display is being enabled.

layoutChildren()method 
override protected function layoutChildren():void

Used to layout the display children of the display. This method is called initially to set the position and size of any child objects and should be called whenever the children need to update their position or size because the layout has changed, for example after the application window has been resized.

This is an abstract method. Override it in your sub-display class and set the position and size of all child display objects here.

reset()method 
override public function reset():void

Used to put the display into it's initial state as it was right after the display has been instantiated for the first time. This method can be called to reset properties and child objects in case the display should be re-used without the need to re-instantiate it.

This is an abstract method. You can override this method in your display sub-class and place any instructions into it that need to be done to reset the display.

start()method 
override public function start():void

Starts the display after it has been initialized. You normally don't call this method manually. Instead the parent screen calls it automatically when the screen is being started.

This is an abstract method. Override this method in your state sub-class and place any instructions into it that need to be done when the state is being started, for example a display might contain animated display children that should start playing after this method has been called.

stop()method 
override public function stop():void

Stops the display if it has been started. You normally don't call this method manually. Instead the parent screen calls it automatically when the screen is being stopped.

This is an abstract method. You can override this method in your display sub-class and place any instructions into it that need to be done to stop the display, for example stopping an animation.

unpauseChildren()method 
override protected function unpauseChildren():void

Used to unpause any child display objects that were paused. Called whenever the paused property of the display is set to false.

This is an abstract method. Override it in your sub-display class and unpause any child display objects here that should be unpaused when the display is being put into unpaused mode.

update()method 
override public function update():void

Updates the display. This method is called automatically by the parent screen before it is being started. It updates any display text that the display might contain and lays out it's display children.

This method can be called if child objects of the display need to be updated, e.g. after localization has been changed or if the display children need to be re-layouted.

updateDisplayText()method 
override protected function updateDisplayText():void

Used to update any display text that the display might contain. Typically any textfield text should be set here with strings from the application's currently used text resources. Called whenever the display's update() method is called.

This is an abstract method. Override it in your sub-display class and set strings from text resources to any text displays if they require it.
