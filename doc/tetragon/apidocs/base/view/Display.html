Classpublic class Display
InheritanceDisplay flash.display.Sprite
Subclasses DummyDisplay, Screen, SplashDisplay

The abstract base class for all display classes, inclusive screens. A Display is a container that can contain any kind and number of other display objects (Sprites, MovieClips, Shapes, Bitmaps etc.) and which is added for display on a screen.

Property Detail
enabledproperty
enabled:Boolean

Determines if the display is enabled or disabled. On a disabled display any display children are disabled so that no user interaction may occur until the display is enabled again. Set this property to either true (enabled) or false (disabled).

The default value is true.

pausedproperty 
paused:Boolean

Determines whether the display is in paused state or not. If paused, any child objects of the display are being paused too, if possible. This property should be used if the display needs to be pausable, for example if it contains any animation that should not play while the application is in a paused state.

The default value is false.

dispose()method 
public function dispose():void

Disposes the display to clean up resources that are no longer needed. A call to this method stops the display and removes any event/signal listeners. You normally don't call this method manually. Instead the parent screen calls it automatically when the screen is being disposed.

If you want to override this method make sure to call super.dispose() at the start of your overriden dispose method.

public function reset():void

Used to put the display into it's initial state like it was right after the display has been instantiated for the first time. This method can be called to reset properties and child objects in case the display should be re-used without the need to re-instantiate it.

update()method 
public function update():void

Updates the display. This method is called automatically by the parent screen before it is being started. It updates any display text that the display might contain and lays out it's display children.

This method can be called if child objects of the display need to be updated, e.g. after localization has been changed or if the display children need to be re-layouted.

updateDisplayText()method 
protected function updateDisplayText():void

Used to update any display text that the display might contain. Typically any textfield text should be set here with strings from the application's currently used text resources. Called whenever the display's update() method is called.

This is an abstract method. Override it in your sub-display class and set strings from text resources to any text displays if they require it.
